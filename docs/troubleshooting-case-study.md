# Разбор инцидента (пример)

## Карточка инцидента

- Incident ID: `INC-NET-2026-001`
- Приоритет: `High`
- Область влияния: плавающие потери пакетов у клиентского трафика в одном регионе
- Время начала: `2026-03-10 11:20 UTC`

## Симптомы

- рост ретрансмитов в сервисных потоках
- жалобы пользователей на нестабильные сессии
- алерты мониторинга по drop на edge-интерфейсе

## Таймлайн расследования

1. Подтвержден scope и список затронутых префиксов.
2. Проверено: control-plane OSPF/BGP в норме.
3. Обнаружены burst-дропы на одном участнике агрегированного uplink.
4. Выявлен рассинхрон настроек LACP/duplex после изменения.
5. Параметры выровнены, выполнен мониторинг на предмет рецидива.

## Корневая причина

Конфигурационный рассинхрон одного физического участника LACP-бандла вызвал асимметричные потери под пиковой нагрузкой.

## Исправление и проверка

- Применен единый профиль интерфейса на обоих концах.
- Сброшены счетчики и проверена стабильность в течение 60 минут.
- Подтверждено восстановление пользовательских сессий.

## Профилактика

- Добавить pre-change проверку консистентности конфигов.
- Включить алертинг по ошибкам каждого LACP-члена.
- Дополнить runbook post-change проверкой бандла.
